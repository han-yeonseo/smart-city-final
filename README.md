## 데이터 처리 과정

### 1. 격자 종사자 (`scripts/01_grid_employment.py`)

- SGIS 100m 격자 종사자 파일 (`라사`, `라바` 구역 CSV) 로드
- 좌표 변환: WGS84 → TM중부원점(EPSG:5186)
- 격자 원점 보정: `sx = -415204`, `sy = 190068`
- 구역 경계로 클리핑
  - 제1판교: 라사 `row 414~441 / col 227~271` → **385개 격자, 23,097명**
  - 청라국제업무지구: 라바 `row 583~609 / col 829~856` → **16개 격자, 26명**
- 결과 저장: `data/pangyo_employment.json`, `data/cheongna_employment.json`

### 2. 인구 및 직주비 (`scripts/02_population.py`)

- SGIS 집계구 인구총괄 파일 로드 (총인구, 성연령별)
- 집계구 shapefile 미확보 → **면적 비례 배분**으로 구역 내 인구 추정
  - 제1판교: `922,202명 × (3.3 / 71.5) ≈ 42,600명`
  - 청라: `1,083,068명 × (1.5 / 154.0) ≈ 10,500명`
- 직주비 = 종사자수 ÷ 상주인구
- 결과 저장: `data/pangyo_population.json`, `data/cheongna_population.json`

### 3. 등시간권 (`docs/data.js`)

- 수도권 지하철 네트워크 + 다익스트라 알고리즘
- 핵심역: 판교역(신분당선) vs 청라국제도시역(공항철도)
- 30분·60분 도달 역 목록 산출
- 결과: `PANGYO_ISO`, `CHEONGNA_ISO` 배열로 저장

## 분석의 한계

- 구역 내 인구는 집계구 shapefile 미확보로 **면적 비례 배분 추정값** 적용
- 청라 종사자 26명은 2020년 업무지구 미개장 상태를 반영하며, 직주비 의미 있는 해석 불가
- 건축물대장은 행정동 단위로 수집되어 정확한 구역 경계와 불일치 가능성 있음
- 교통 접근성은 지하철 기반으로만 산출 (버스·자동차 미포함)
